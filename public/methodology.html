Sources & Methodology
How this site is built

What we count as evidence, where the data comes from, and how automated updates work.

Principle

Separate claims from sources. If a page states a fact, it should be traceable to a source (official portal, affidavit, court order, or clearly named reporting). This page documents the default sources and the update rules used across the site.

What “verification” means here

Verification is specific to each tool/page, but the common rule is consistency: we only present a detail as verified when it is attributable to a named primary source or a named secondary source with a stable URL.

  • Election results: candidate names appear only when confirmed by official results pages or named sources; otherwise a seat may show party attribution without names.
  • Corruption tracker updates: timeline entries are only added when there is either an official enforcement agency press release or a clear multi-article signal (2+ in 24 hours) that indicates meaningful movement.
  • Accountability tracker: statuses are descriptive labels; the aim is to change status only on official orders or independently verifiable actions.

Automated updates (every 6 hours)

The site includes an automated update job that refreshes news and metadata on a schedule. It writes these files in /data:

  • data/news.json (rolling headlines per case, refreshed each run)
  • data/cases.json (long-term timelines; updated only when rules are met)
  • data/meta.json (timestamps such as lastUpdated/autoChecked)
  • data/pledges.json (adds latest pledge headline; auto-marks delayed pledges when deadlines pass)
Primary inputs
  • Google News RSS search queries (per case keywords).
  • Enforcement Directorate press releases (enforcement.gov.in), filtered for Bengal-relevant text.
  • CBI press releases (cbi.gov.in), filtered for Bengal-relevant text.

Election results scraping (scheduled)

The constituency results tool can auto-fill entries that are still marked unverified by scraping the official ECI constituency pages for West Bengal and updating the table data in place.

ECI pattern

results.eci.gov.in constituency pages follow a predictable URL pattern for West Bengal (state code S25). The scraper reads the winner row (status “Won”), normalises party labels, and computes the winning margin.

Editorial independence

This is intended as a civic education and transparency project. It is not affiliated with any party, campaign, or government body. Where the site has an argument or point of view, it should be clearly distinguishable from factual reporting.

Corrections

If you find a factual error, the fastest way to get it fixed is to cite the correct source and link to the page/section where the issue appears. Corrections should prefer primary sources (ECI, affidavits, official orders) over screenshots or social posts.

Transparency

This site is a static build that can be audited: the pages are plain HTML and the update scripts are plain Node scripts. Data files are committed to the repository so changes are reviewable over time.
